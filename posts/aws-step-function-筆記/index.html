aws step function 筆記

Jan 16, 2021 2 分鐘閱讀 程式awsaws-step-function

最近工作用到一些工具，使用到AWS step function , 因此在這裡也筆記一下… 也在公司後端組例會分享了一下（以下就是我分享的PPT ）：

https://www.slideshare.net/ssusereb2ee2/aws-stepfunction

其實我覺得我用的情境很簡單，只是用Map 的方式啟動lambda . 這個 lambda 就是我用來處理下載與上傳到s3指定位置… 說真的應用的情境真的很不多… 還有更多著墨的空間。

另外，自己同時也針對此寫了兩個版本，用SAM 和 用 CDK 的版本…

一、CDK 的版本

 1import * as cdk from '@aws-cdk/core';
 2import * as lambda from "@aws-cdk/aws-lambda"
 3import * as stepfunctions from "@aws-cdk/aws-stepfunctions"
 4import * as tasks from "@aws-cdk/aws-stepfunctions-tasks"
 5import * as logs from "@aws-cdk/aws-logs"
 6import * as s3 from "@aws-cdk/aws-s3"
 7import * as ec2 from "@aws-cdk/aws-ec2"
 8import * as dotenv from 'dotenv';
 9
10export class CdkLambdaStack extends cdk.Stack {
11  constructor(scope: cdk.Construct, id: string, props?: cdk.StackProps) {
12    super(scope, id, props);
13
14    // 將裡面比較敏感的資訊用env 包起來, 注意後面的path 要正確
15    dotenv.config({path:__dirname+'/../.env'})
16    
17    // 我要上傳音檔的S3 目標 arn:aws:s3:::test 為虛構（我忘了把這個也包env了哈哈）
18    const bucket = s3.Bucket.fromBucketArn(this,"test","arn:aws:s3:::test")
19
20    // 負責前面呼叫step function 的 lambda
21    const downloadAudioLambda = new lambda.Function(this, "downloadAudioLambda", {
22      runtime: lambda.Runtime.NODEJS_12_X,
23      timeout: cdk.Duration.seconds(25),
24      handler: "index.handler",
25      code: lambda.Code.fromAsset("lambda/downloadAudioLambda")
26    });
27
28    bucket.grantPut(downloadAudioLambda)
29
30    const downloadAudioJob = new tasks.LambdaInvoke(this,'Calllambda',{
31      lambdaFunction: downloadAudioLambda,
32      outputPath: "$.Payload"
33    })
34
35    const map = new stepfunctions.Map(this, 'ExampleMapState');
36    map.iterator(downloadAudioJob);
37
38    const logGroup = new logs.LogGroup(this, 'StepFunctionLogs')
39
40    const stateMachine = new stepfunctions.StateMachine(this, 'StateMachine', {
41        definition: map,
42        logs: {
43          destination: logGroup,
44          level: stepfunctions.LogLevel.ERROR
45        }
46    });
47
48    const testVpc = ec2.Vpc.fromLookup(this,"vpc-dev",{
49      vpcId: process.env.VPCID
50    });
51
52    const processorLambda = new lambda.Function(this, "processorLambda", {
53      runtime: lambda.Runtime.NODEJS_12_X,
54      handler: "index.handler",
55      timeout: cdk.Duration.seconds(25),
56      code: lambda.Code.fromAsset("lambda/processor"),
57      vpc: testVpc,
58      environment: {
59        ENDPOINT: process.env.ENDPOINT ?? 'localhost',
60        DATABASE: process.env.DATABASE ?? 'db',
61        DBUSERNAME: process.env.DBUSERNAME ?? 'root',
62        PASSWORD: process.env.PASSWORD ?? 'password',
63        NODE_ENV: process.env.NODE_ENV ?? 'test',
64        statemachine_arn: stateMachine.stateMachineArn
65      }
66    });
67
68    stateMachine.grantStartExecution(transferLambda)
69  }
70}

總之，上面我就是用CDK先創建我的lambda , 然後把那個要放到state machine 的建立"task", 給予我另外一個lambda 有 startExecution 的權限…. 簡單完成！

二、SAM 的版本

總之，有些原因，我另外又學習怎麼用SAM製作 state machine XDD

 2Transform: AWS::Serverless-2016-10-31

 4  download audio file from huaxi to trigger audio transcoder

其實說真的CDK 和 SAM 沒有多大差別，只是CDK你可以用比較程式化的去做那個state machine language （就是sam 裡面要包的那個json 啦！），像我，實在懶得去構想那個json 怎麼寫（啊我就不是JSON工程師啊～），所以先用CDK 產生state machine , 然後上AWS控制台上面把那一串json 抓下來，放到我的sam 這裏… 整理一下，CDK detroy 一下，sam 的template.yaml 調整一下，一個下午搞定啦！（不過我好像忘了在sam 裡面宣吿log 去接state machine 啦 XDDD 之後再研究吧！）
